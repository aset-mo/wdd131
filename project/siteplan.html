Website Planning Document

Project Title: Andes Trekker

Site Name

Andes Trekker

The name reflects the core focus of the website: trekking routes in the Andes Mountains. It’s clear, concise, and memorable for outdoor and adventure enthusiasts.

Optional domain availability: andestrekker.netlify.app or andestrekker.github.io

Site Purpose

The purpose of Andes Trekker is to provide up-to-date information on trekking routes in the Andes. The site will offer route recommendations, difficulty levels, trail maps, equipment checklists, and a form for subscribing to newsletters and guided hiking events.

Scenarios

  • What trekking routes are suitable for beginners in the Andes?
  • Where can I download a detailed map and elevation profile of a specific route?
  • How can I register for upcoming guided hiking tours?

Target Audience

Outdoor enthusiasts, hikers, travelers visiting South America, and people researching trekking options in the Andes.

Color Scheme

Color Usage HEX
Forest Green Headings, buttons #259857
Light Gray Backgrounds, input fields #F5F5F5
Charcoal Black Body text #333333

Typography

  • Montserrat: Used for all headings and navigation links.
  • Roboto: Used for paragraph/body text and form inputs.

Fonts will be loaded using Google Fonts.

Pages

Your website will contain at least 3 main pages:

  1. Home (index.html) — Hero image, introduction, call to action, route highlights.
  2. Routes (routes.html) — List of trekking routes (generated from JavaScript array), with sorting/filtering features.
  3. Contact (contact.html) — HTML form for newsletter registration or inquiries.
  4. References (references.html) — Cites all external sources (text, images).

HTML Form

  • Located in contact.html
  • Required fields: name, email, message
  • Optional checkbox for newsletter subscription
  • Use required, type, and pattern attributes for validation

Image Use

All images will use:

<img src="img/path.jpg" alt="Trail image" loading="lazy">

Footer

Repeated across all pages

  • Includes copyright
  • Link to references.html

Example:

<footer>
  <p>&copy; 2025 Andes Trekker. All rights reserved.</p>
  <a href="references.html">References</a>
</footer>
